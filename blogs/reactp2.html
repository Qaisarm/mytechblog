React for Beginners

Part 2: A syntax extension to JavaScript: JSX

During learning React you may have seen a funny syntax that looks like HTML code inside of the React application. That is JSX, a syntax extension to JavaScript.

This syntax is a combination between HTML and JavaScript. The JSX elements are used to produce the React elements.

Why JSX is used?

In traditional applications, the view (HTML) is seperated from the logic (JavaScript). So, the React is built upon the idea that the data within the application is related to the logic which renders components. The React couples the markup and the logic together inside of components.

The use of JSX is not mandatory for creating the React applications, but it is extremely useful.

JSX Expressions

JSX expressions are used to pass variables to elements and properties, evaluate boolean expressions, and more! We can use curly braces ({}) to create expressions in JSX.

Attributes

The string attributes can be specified, just like it is done in HTML, with quotes or double quotes.

const header = < header className=" myHeader">;

To evaluate the JavaScript expression, curly braces can also be used. There is no need to wrap the curly braces in quotes; one can use either quotes (strings) or curly braces (expressions), but can not both.

const profilePicture = < img src = {user.picture} alt = "Me!" />;

Conditional Classes

To set a conditional classes that is based on a variable is very useful in React. Such classes allow us to say: if this condition is true, then add these classes to the component, otherwise add these classes.

One can use it to hide and show the dialogs or modals when the visibility is toggled, for example. Below is the basic syntax.

< h1 className= {condition ? true condition : false condition}>;

JSX is an Expression Too

After compilation of the code, the JSX expressions becomes a regular JavaScript function that calls and evaluate to JavaScript objects. This means that JSX can be used inside of if statements and for loops, assign it to variables, accept it as arguments, and return it from functions:

function getGreeting(user) {
if (user) {
return < h1>Hello, {formatName(user)}!;
}
return < h1>Hello, Stranger.;
}

In the above code, we return the user,s name if they're logged in and otherwise just return Hello Stranger!.

We can also re-factor the code to make it a bit more succinct.

function getGreeting(user) {
return user.isAuthenticated ?< h1> Hello, {formatName(user)}! :< h1>Hello Stranger!.;
}
}

function getGreeting(user) {
return < h1> Hello, {formatName(user).isAuthenticated ?formatName(user): 'Stranger!'.;

Self-Closing Tags

If an element has no children, it is a self-closing element. This means you don't need the corresponding end tag.

So instead of writing < element> < /element> , you would write < element /> .

JSX is pretty simple to use once you get your hands dirty with it.
