React for Beginners

Part 1: Introducion and Project setup

React is one of the usefull JavaScript library that is used to build web applications. This library is Declarative , which allows you to control the flow and the state of your application by allowing you to describe what it should look like. This is in contrast to Imperative libraries, which describe what your web application should do.

React was developed by Facebook in 2011 and is one of the most widely used JavaScript frameworks.

React skillsets are in demand and many employers are looking for such candidates, but the number of persons with required skill set is lower. Learning React can be overwhelming. There are so many new concepts to familiarize yourself with and many of the tutorials can be hard to follow. So let's break down key concepts from React. I am not an expert on React, so if you find mistakes, feel free to leave a private note or comment below!

Getting Started

The easiest way to create a new single-page application in React is to use create-react-app. This is a command line interface (CLI) which allows you to quickly scaffold a React application .

This scaffold code includes all of the development and workspace configuration so you can easily get started.

To use the React CLI, first you need to install it globally. You can use npm or Node Package Manager to run the installation.

Create A React Application In A New Folder

Before creating a React application you'll need to have Node >= 14.0.0 and npm >= 5.6 on your machine.

To create a project, run the following commands:


npx create-react-app my-app
cd my-app
npm start

Note that npx on the first line is not a typo â€” it's a package runner tool that comes with npm 5.2+.

Create A React Application In An Existing Folder

cd my-app
npx create-react-app .
npm start

If you do not have integrated development environment (IDE) installed on your device then CodeSandbox.io is another way to play around with the React code. This wonderful web-based tool allows you to experiment and download the source-code afterwords.

Create-React-App

Let's examine the application structure when you create the React app with create-react-app.

In your terminal or command line, run create-react-app my-app. A new folder will be created in your current directory with the starter React app.

Once that's done, cd my-app and run npm install or yarn install. This will download all of the dependencies that are needed for the starter application.

Now, open the application in an IDE of your choice. I am using Visual Studio Code here.

Line 1: Image file of logo is imported, the displays the logo image on UI

Line 2: Import the CSS file of the App.

Line 3: Create the functional component of the App.

There are two different types of components in React: functional and class components.
Functional Components are declared with the function keyword and accept props as an argument. They simply return valid React element.
Class Components are declared with the class keyword and extend React.Component or just Component if you have imported it using the destructured format.
Both functional and class components are logically equivalent (class just provides more functionality).

Line 23: Export the App component.

Multiple named exports in a single JavaScript file can be used such as :

export class FirstClass {}
export class SecondClass {}

These exports must be imported using their exact names:

import { FirstClass, SecondClass } from './App';

When we use a defaultexport, we can do the following:

export default class FirstClass {}

And import it this way:

import FirstClass from './App';

There can only be one default export per JavaScript file. In React, it's good practice to export one component per file, so export default is the common way to export your components.
